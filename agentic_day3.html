Memory & State Management 🧠

Short-term vs Long-term · Buffers · Vector Stores · State Persistence
🎯 Use-case: Personal Shopping Assistant that remembers your style, size & budget

What & Why

What is Memory in Agentic AI? The ability of an agent to retain information across interactions — both within a single conversation (short-term) and across sessions (long-term).

Why crucial? Without memory, an agent treats every query as first contact. For a shopping assistant, remembering “prefers cotton shirts, size L, budget $50” is the difference between a generic bot and a personalized concierge.

Business Impact

  • ✅ Increases user engagement & retention
  • ✅ Reduces repetitive questioning
  • ✅ Enables context-aware recommendations
  • ✅ Builds trust through continuity

How & Where

How memory is implemented: Using conversation buffers, vector databases, and summary techniques. Where memory lives: in-memory (session) or persistent (database/vector store).

  • 📍 Short-term memory – sliding window buffers, in-context tokens
  • 📍 Long-term memory – vector DB (Chroma/FAISS) storing embeddings of user preferences
  • 📍 Summary memory – compress old conversation into condensed notes
  • 📍 State persistence – saving agent state (user ID, preferences) to disk or cloud
When to use Vector Stores: For open-ended user profiles, product affinity, and cross-session retrieval. FAISS/Chroma allow semantic search over past interactions.

Conversation Buffer Memory

Stores recent dialogue turns. ConversationBufferWindowMemory keeps last K messages to avoid token overflow.

from langchain.memory import ConversationBufferWindowMemory

memory = ConversationBufferWindowMemory(k=3, return_messages=True)
memory.save_context({"input": "I like blue shoes"}, 
                    {"output": "Got it, blue preference saved"})
print(memory.load_memory_variables({}))

✔️ Best for: ongoing chat within a single session.

Vector Store Long-Term Memory

Embed user preferences and retrieve similar memories using semantic search. Ideal for cross-session memory.

from langchain.embeddings import OpenAIEmbeddings
from langchain.vectorstores import FAISS
import pickle

# store preferences as vector docs
prefs = ["user prefers size M", "likes minimalist design", "budget under $100"]
vector_db = FAISS.from_texts(prefs, OpenAIEmbeddings())
# later retrieval
similar = vector_db.similarity_search("budget limit", k=1)
print(similar[0].page_content)  # 'budget under $100'

Summary Memory

Periodically summarizes conversation history to maintain big-picture context while reducing token usage.

from langchain.memory import ConversationSummaryMemory
summary_memory = ConversationSummaryMemory(llm=llm)
summary_memory.save_context({"input": "I need a red dress"}, 
                            {"output": "Searching red dresses"})
# produces summary: "User wants a red dress"

State Persistence

Save memory objects (buffers, vector indices) to disk/Redis to survive restarts.

# Save FAISS index
vector_db.save_local("shopping_assistant_index")
# Load again
restored_db = FAISS.load_local("shopping_assistant_index", embeddings)

🛍️ Use-Case: Personal Shopping Assistant

An agent that remembers user preferences (size, color, budget, style) across sessions using FAISS vector database + persistent storage. The assistant can store and retrieve past likes/dislikes, and answer with personalized recommendations.

User: "I love leather jackets" Vector DB Store Semantic Retrieval Personalized Reply

Complete Implementation Example (Python + FAISS + LangChain)

import os
from langchain.memory import ConversationBufferMemory
from langchain.embeddings import OpenAIEmbeddings  # or HuggingFace
from langchain.vectorstores import FAISS
from langchain.chains import ConversationalRetrievalChain
from langchain.chat_models import ChatOpenAI
import pickle

class ShoppingAssistant:
    def __init__(self, user_id):
        self.user_id = user_id
        self.embeddings = OpenAIEmbeddings()
        self.llm = ChatOpenAI(model="gpt-3.5-turbo", temperature=0.2)
        # Try to load existing vector DB for this user
        self.vector_db = self._load_vector_db()
        self.conversation_memory = ConversationBufferMemory(
            memory_key="chat_history", return_messages=True
        )
        
    def _load_vector_db(self):
        path = f"memory_db_{self.user_id}.faiss"
        if os.path.exists(path):
            return FAISS.load_local(path, self.embeddings)
        else:
            # Initialize empty with dummy text (will add later)
            return FAISS.from_texts(["init"], self.embeddings)
    
    def _save_vector_db(self):
        self.vector_db.save_local(f"memory_db_{self.user_id}.faiss")
    
    def remember_preference(self, preference_text):
        """Store user preference in long-term memory (vector store)"""
        self.vector_db.add_texts([preference_text])
        self._save_vector_db()
        return f"📝 Remembered: '{preference_text}'"
    
    def retrieve_preferences(self, query):
        """Search related preferences from past sessions"""
        docs = self.vector_db.similarity_search(query, k=3)
        return [doc.page_content for doc in docs]
    
    def chat(self, user_input):
        # Retrieve relevant long-term memories
        long_term_memories = self.retrieve_preferences(user_input)
        context = "\n".join(long_term_memories)
        # Build a prompt that includes both memory sources
        prompt = f"User shopping profile (from history): {context}\nUser: {user_input}\nAssistant:"
        response = self.llm.predict(prompt)
        # Optionally, extract new preferences from this interaction (simplified)
        if "like" in user_input.lower() or "prefer" in user_input.lower():
            self.remember_preference(user_input)
        return response

# Example usage across sessions
assistant = ShoppingAssistant("alice123")
print(assistant.chat("I need a summer dress, size S, under $60"))
print(assistant.chat("Show me blue sneakers"))
# Next day / new session – load same user_id, memory persists!
assistant2 = ShoppingAssistant("alice123")
print(assistant2.chat("What was my size preference?"))  # retrieves size S
Key takeaways: The agent uses FAISS to remember long-term preferences across different sessions. You can combine with ConversationBufferMemory for short-term dialogue flow.

Where to use each type

  • Buffer window → real-time customer support chat
  • Vector store → long-term user profile, product recommendations, cross-session memory
  • Summary memory → very long conversations (e.g., legal or medical advisory)
  • Hybrid → shopping assistant: buffer for current cart + vector DB for past tastes
