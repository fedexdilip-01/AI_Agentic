📐 LangGraph Architecture

State Machine for Conversations

Graph Nodes: classify_intent → (rag_query | api_call | human_handoff) → final response. Conditional edges route based on confidence thresholds.

🔹 classify_intent: identifies order, refund, faq, or escalation needs
🔹 rag_query: retrieves from FAISS (policy docs, FAQs)
🔹 api_call: order status / create refund ticket
🔹 human_handoff: confidence < 0.7 → transfer to live agent

Why? Graph-based flow ensures reliability, debuggability, and deterministic handoffs.

⚙️ RAG + FAISS + API

Knowledge & Transactions

FAISS index: stores embeddings of FAQ documents (returns, shipping, policies). Order API: mocks database lookup for order status. Refund tickets: generates ticket IDs for returns.

How it works: User query → embed → FAISS similarity search → generate answer from top chunks. Confidence scoring drives escalation.

🧑‍💼 Human Handoff & Deployment

Production Readiness

Handoff trigger: if intent confidence < 0.7 OR user explicitly requests human → agent creates a support ticket and notifies human queue.

Deployment: FastAPI backend, Streamlit frontend. This demo simulates the full graph with tracing, FAISS retrieval, and handoff logic.
