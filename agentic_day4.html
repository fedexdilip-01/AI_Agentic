📐 Reasoning Paradigms

CoT · ToT · Plan-and-Execute

Chain-of-Thought (CoT): step-by-step reasoning before answering. Tree-of-Thoughts (ToT): explores multiple reasoning paths, backtracks. Plan-and-Execute: decompose high-level goal into subtasks → execute each step → aggregate results.

// Plan-and-Execute example
1. Fetch paper from arXiv
2. Summarize each section
3. Extract key findings
4. Compare with past papers (vector similarity)

Why? Complex tasks (research analysis) need decomposition, error handling, and dynamic replanning if steps fail.

⚙️ Error Recovery & Replanning

Robust Agentic Workflows

Planners handle failures: if API fails → retry/fallback; if extraction ambiguous → re-query. Replanning adjusts subsequent steps based on intermediate results. The agent observes outcomes and adapts.

Where? Research summarizers, automated code generation, scientific workflows, travel planning, etc. Multi-step reasoning is critical for real-world agents.
