🧰 Day 2: Tool Use & Function Calling

Designing tools, structured outputs, tool selection strategies — applied to a Flight & Hotel Booking Assistant

⚙️ Core Concept

What is Tool Use & Function Calling?

Tool use enables LLM agents to interact with external systems: APIs, databases, calculators, or business logic. Function calling is the structured way LLMs decide which tool to invoke, with what arguments, based on user intent.

Why? LLMs alone cannot fetch live data, perform bookings, or execute actions. Tool calling bridges the gap between reasoning and real-world impact.

📌 Example: User: "Book a flight to NYC on June 10" → Agent calls search_flights(city="NYC", date="2025-06-10") → presents options → calls book_flight(flight_id).
🔨 How & Design

Designing Tools & Output Parsing

Tool design: define name, description, input schema (JSON). Tools can be APIs (e.g., flight search), calculators, search engines. Structured output parsing ensures the agent returns consistent data (e.g., Pydantic/JSON schema).

Tool selection strategies: rule-based (if keywords match), LLM-driven (few-shot prompting), or semantic router. In production, hybrid approaches combine intent classification + dynamic tool selection.

🔧 Flight tool, hotel tool, booking tool: each with typed parameters. Agent selects the correct tool by analyzing the user prompt.
🧭 Use-case: Travel Assistant

Flight & Hotel Booking Assistant

Where & Why: Travel booking requires real-time search, multi-step interactions, and atomic booking actions. Agent must orchestrate: search flights → search hotels → reserve flight ID → confirm booking.

Task implementation: Three tools: search_flights(city, date), search_hotels(city), book_flight(flight_id). Agent infers user intent, calls the correct tool, and presents structured results. Handles multi-turn conversation.
